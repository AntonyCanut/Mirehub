Demarrage rapide Getting started

Mirehub est un IDE macOS natif qui reunit 10 outils de developpement dans une seule fenetre : Terminal, Claude AI, Git, Kanban, NPM, TODO Scanner, Statistiques, Prompts, Database Explorer et API Tester. Aucune configuration n'est necessaire. Les mises a jour sont automatiques. Mirehub is a native macOS IDE that combines 10 development tools in a single window: Terminal, Claude AI, Git, Kanban, NPM, TODO Scanner, Statistics, Prompts, Database Explorer and API Tester. No configuration needed. Updates are automatic.

Installation Installation

  1. Telechargez le fichier .dmg depuis la page des releases GitHub Download the .dmg file from the GitHub releases page
  2. Ouvrez le .dmg et glissez Mirehub dans votre dossier Applications Open the .dmg and drag Mirehub to your Applications folder
  3. Lancez Mirehub et pointez vers votre premier projet Launch Mirehub and point to your first project

Configuration requise System requirements

  • macOS 12+ (Monterey)
  • Apple Silicon & Intel
  • ~150 MB
  • Claude Code installe (pour les fonctionnalites IA) Claude Code installed (for AI features)

Integration Claude AI Claude AI integration

Le Kanban de Mirehub est propulse par l'IA. Chaque ticket peut etre envoye directement a Claude Code qui le resout automatiquement. Voici le fonctionnement complet : Mirehub's Kanban is AI-powered. Every ticket can be sent directly to Claude Code which resolves it automatically. Here is the complete workflow:

  1. L'utilisateur cree un ticket (T-01, T-02...) avec titre, description, priorite et labels The user creates a ticket (T-01, T-02...) with title, description, priority and labels
  2. Cliquez sur le bouton "Envoyer a Claude" sur n'importe quel ticket Click the "Send to Claude" button on any ticket
  3. Mirehub ouvre automatiquement un onglet terminal avec Claude Code Mirehub automatically opens a terminal tab with Claude Code
  4. Claude recoit un fichier prompt (.mirehub/.kanban-prompt-{id}.md) avec les details du ticket Claude receives a prompt file (.mirehub/.kanban-prompt-{id}.md) with the ticket details
  5. Des variables d'environnement sont definies : MIREHUB_KANBAN_TASK_ID, MIREHUB_KANBAN_FILE Environment variables are set: MIREHUB_KANBAN_TASK_ID, MIREHUB_KANBAN_FILE
  6. Claude travaille sur la tache de maniere autonome Claude works on the task autonomously
  7. A la fin, Claude met a jour le fichier JSON du kanban When done, Claude updates the kanban JSON file
  8. Un hook (kanban-done.sh) detecte la completion et deplace le statut du ticket A hook (kanban-done.sh) detects completion and moves the ticket status
  9. Le tableau Kanban se met a jour en temps reel via file watching (fs.watch) The Kanban board updates in real-time via file watching (fs.watch)

Si Claude a une question, il passe le ticket en statut PENDING avec un champ question. Si Claude echoue, il passe le ticket en FAILED avec un champ erreur. If Claude has a question, it sets the ticket to PENDING status with a question field. If Claude fails, it sets the ticket to FAILED with an error field.

Planification automatique Automatic scheduling

Quand une session Claude termine un ticket, elle cherche automatiquement la tache suivante. L'ordre de priorite est le suivant : When a Claude session finishes a ticket, it automatically looks for the next task. The priority order is:

  1. Reprendre les taches WORKING en cours d'abord Resume WORKING tasks first
  2. Puis prendre la prochaine tache TODO par ordre de priorite Then pick the next TODO task by priority order

Pipeline entierement autonome : creez vos tickets, Claude les traite un par un sans intervention. Fully autonomous pipeline: create your tickets, Claude handles them one by one without intervention.
